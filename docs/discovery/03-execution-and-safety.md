## Current guardrail summary

The codebase has practical guardrails but not product-level authority boundaries. Examples include Zod request/cell schemas, filename validation, app relative-path checks, process registries, dependency validation, and decode-after-generate checks. These are useful but do not answer: which cells are allowed to mutate code, which validators are authoritative, what capabilities were approved before execution, or what evidence proves a run is replayable.

## Effect-TS implication

Every side effect above maps naturally to a service boundary. The core product reason to consider Effect is not aesthetic typed FP; it is to make effect requirements explicit, injectable, logged, replayable, and deny-by-default for agent execution.
